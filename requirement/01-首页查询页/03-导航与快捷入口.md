# 1.3 用户在首页/查询页登录/注册

## 1.3.1 用户需要登录12306账户
**Scenario: 用户需未登录12306账户**
- Given 用户在**首页/查询页**
- When 用户点击页面顶部的“**登录**”按钮
- And  用户未登录12306账户
- Then 系统跳转至登录页 (`/login`)

**Scenario: 用户已登录12306账户**
- Given 用户已登录12306账户
- When 用户查看页面顶部导航栏
- Then 顶部导航栏显示“您好，[用户名]”和“退出”按钮，不显示“登录”和“注册”按钮

**Scenario: 用户点击退出登录**
- Given 用户已登录12306账户
- When 用户点击顶部导航栏的“**退出**”按钮
- Then 系统弹出确认框提示“确定要退出登录吗？”
- When 用户点击确认
- Then 系统执行退出操作，清除用户会话，并刷新当前页面

## 1.3.2 用户需要注册12306账户
**Scenario: 用户需未登录12306账户**
- Given 用户在**首页/查询页**
- When 用户点击页面顶部的“**注册**”按钮
- Then 系统跳转至注册页 (`/register`)

# 1.4 用户在首页/查询页需前往个人中心

## 1.4.1 通过顶部导航前往
**Scenario: 用户已登录12306账户**
- Given 用户已登录12306账户
- When 用户点击顶部导航栏的“**我的12306**”链接或用户名链接

**Scenario: 用户未登录点击"我的12306"**
- Given 用户未登录12306账户
- When 用户点击顶部导航栏的“**我的12306**”链接
- Then 系统跳转至登录页 (`/login`)，引导用户先登录

# 1.5 主导航栏交互与跳转

## 1.5.1 导航菜单悬停效果
**Scenario: 用户鼠标悬停在“车票”菜单上**
- Given 用户在任意页面
- When 用户将鼠标悬停在主导航栏的“**车票**”菜单项上
- Then 系统延迟显示“车票”二级下拉菜单（包含购买、变更、更多三列子项）
- When 用户将鼠标移出“车票”菜单及下拉区域
- Then 系统延迟隐藏下拉菜单

**Scenario: 用户鼠标悬停在“商旅服务”菜单上**
- Given 用户在任意页面
- When 用户将鼠标悬停在主导航栏的“**商旅服务**”菜单项上
- Then 系统延迟显示“商旅服务”二级下拉菜单（包含餐饮•特产等子项）

## 1.5.2 导航链接跳转
- **首页**: 点击跳转至首页 (`/`)。
- **车票**: 点击跳转至车次列表页 (`/train-list`)。
  - **单程**: 点击下拉菜单中的“单程”，跳转至 `/train-list?type=single`。
- **商旅服务 -> 餐饮•特产**: 点击跳转至餐饮服务页 (`/catering`)。
- **其他菜单项**: 目前为占位符链接 (`#`)，点击无实际跳转。

## 1.6.1 首页中部快捷服务

- Then (当前实现) 按钮可点击，但暂无具体跳转逻辑（预留接口）。

## 1.6.2 底部信息发布Tab切换
**Scenario: 用户切换信息发布Tab**
- Given 用户在首页底部信息发布区域
- When 用户点击“最新发布”、“常见问题”或“信用信息”Tab
- Then 内容区域即时切换显示对应的内容列表
  - **最新发布**: 显示公告列表。
  - **常见问题**: 显示常见问题分类列表。
  - **信用信息**: 显示失信人员公示空状态。
