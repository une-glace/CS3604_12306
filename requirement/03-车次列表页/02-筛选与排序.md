# 3.2 筛选与排序需求

## 3.2.1 筛选条件 (`FilterConditions`)
支持多维度组合筛选，前端本地过滤 `trains` 数据。筛选区域支持展开/收起。

1. **日期导航**:
   - 显示未来15天的日期条。
   - 选中日期高亮，显示日期和星期。

2. **车次类型**:
   - GC-高铁/城际
   - D-动车
   - Z-直达
   - T-特快
   - K-快速
   - 其他
   - 复兴号
   - 智能动车组

3. **出发/到达车站**:
   - **出发车站**: 动态提取当前结果集中出现的所有出发车站。
   - **到达车站**: 动态提取当前结果集中出现的所有到达车站。
   - 支持“全部”选择和多选。

4. **发车时间**:
   - 下拉选择：
     - 00:00-24:00 (默认)
     - 00:00-06:00
     - 06:00-12:00
     - 12:00-18:00
     - 18:00-24:00

5. **席别**:
   - 商务座
   - 优选一等座
   - 一等座
   - 二等座
   - 一等卧
   - 二等卧
   - 软卧
   - 硬卧
   - 硬座

## 3.2.2 排序逻辑
列表表头支持点击排序，点击切换升序/降序/默认状态：

1. **发车时间** (默认排序)
   - 按 `fromTime` 排序。
2. **到达时间**
   - 按 `toTime` 排序。
3. **历时**
   - 解析 `duration` 字符串（如 "4小时18分" 或 "04:18"）转换为分钟数进行排序。
4. **车次**
   - 按 `trainNo` 字符串排序。

## 3.2.3 交互逻辑
- **实时过滤**: 勾选筛选条件后，列表即时更新。
- **全选/反选**: 各筛选行提供“全部”按钮，点击可清除该行所有勾选（即显示全部）。
- **折叠/展开**: 提供“筛选 ▴/▾”按钮，可折叠除日期和车次类型以外的高级筛选条件。
- **空状态**: 当筛选结果为空时，显示“很抱歉，按您的查询条件，当前未找到...”，并推荐中转换乘。
