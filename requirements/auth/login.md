1. 登录页面（/login）

    1.1 页面布局

        1.1.1 整体布局
            - 页面背景为浅灰色整体容器，顶部为蓝色品牌区域，下方为轮播背景叠加登录卡片
            - 页面水平居中，内容最大宽度与首页保持一致，高度自适应浏览器窗口

        1.1.2 顶部品牌区域
            - 左侧 Logo：使用图片"/铁路12306-512x512.png"，尺寸约48px×48px，左侧对齐
            - 文本信息：
              - 上行文字："中国铁路12306"，字号约20px，字体加粗，颜色为深灰
              - 下行文字："12306 CHINA RAILWAY"，字号约12px，颜色为次要灰
            - 品牌文本与 Logo 水平排列，垂直居中
            - 品牌区域右侧展示欢迎语："欢迎登录12306"

        1.1.3 主体区域（轮播 + 登录表单）
            - 背景为全宽轮播图区域，高度约600px，使用Carousel组件自动轮播两张宣传图片
            - 轮播图片填充模式为cover，定位在左侧中心位置，隐藏切换箭头，仅保留自动轮播效果
            - 右侧叠加登录表单面板，采用绝对定位：
              - 距右侧约40px，距顶部约36px
              - 面板宽度约380px，背景为白色
              - 外层描边为1px 实线 #8bb4ff，无圆角，无额外阴影
            - 登录表单内容整体放置在该面板内部，分为顶部选项卡和下方卡片内容两部分

        1.1.4 登录轮播区域
            - 轮播图片数据：
              - 图片1：src="/banner-login-2.jpg"，标题"登录轮播一"
              - 图片2：src="/login-banner-1.jpg"，标题"登录轮播二"
            - 自动播放，轮播间隔约5秒
            - 图片填充模式为cover，确保在不同屏幕下保持完整展示

        1.1.5 选项卡区域
            - 位置：登录面板顶部，宽度与面板一致（约380px），高度约78px
            - 结构：
              - 包含两个按钮形式的选项卡："账户登录"、"扫码登录"
              - 二者水平排列，中间通过竖线"|"分隔
              - 选项卡容器底部有一条1px 实线 #eee 的分割线
            - 未选中态（普通状态）：
              - 文本颜色为#666（深灰）
              - 字号约18px
              - 背景透明，无边框
              - 垂直方向padding约8px，上下留白
            - 选中态：
              - 文本颜色为#2a78f6（蓝色）
              - 字体加粗（font-weight: 600）
              - 选项卡底部出现2px 实线 #2a78f6 的下划线，强调当前激活的登录方式
            - 鼠标交互：
              - 鼠标悬停时保持按钮形态，点击切换activeLoginTab状态并更新下方内容

        1.1.6 页脚区域
            - 底部显示两行版权信息，内容参考首页
            - 文本居中，颜色为浅灰，字号约12px

    1.2 账户登录模式

        1.2.1 表单结构
            - 表单所在卡片：
              - 使用 login-card 容器包裹，内部上下左右padding约为24px、24px、18px、24px
              - 表单宽度填满卡片可用区域
            - 字段排布：
              - 表单内容自上而下居中排列
              - 字段顺序：
                - 用户名/邮箱/手机号输入框
                - 密码输入框
                - "立即登录"按钮
                - 底部链接区域（忘记密码、注册入口）
                - 最底部服务说明文字区域（通过分割线与表单主体区分）
            - 间距：
              - 单个表单项（form-group）之间竖直间距较紧凑，每项之间约6px
              - 按钮与底部链接区域之间保留少量垂直间距，保证整体紧凑但不拥挤

        1.2.2 用户名/邮箱/手机号输入框
            - 类型：文本输入框
            - 占位符："用户名/邮箱/手机号"
            - 输入内容实时更新内部状态formData.username
            - UI 样式：
              - 输入框宽度约320px，高度约44px
              - 左右内边距约16px，文本字号14px
              - 默认边框为1px 实线 #e0e0e0，背景为白色
              - 获取焦点时：
                - 边框颜色变为#2a78f6（蓝色）
                - 外侧出现约2px 的浅蓝色发光描边（box-shadow: rgba(42,120,246,0.15)）
            - 校验规则：
              - 允许两种形式：
                - 用户名：
                  - 正则：^[A-Za-z]{1}([A-Za-z0-9]|[_ ]|[A-Za-z0-9_ ]){0,29}$
                  - 以字母开头，总长度不超过30字符
                  - 允许字母、数字、空格和下划线
                - 邮箱：
                  - 正则：^[^\s@]+@[^\s@]+\.[^\s@]+$
              - 输入为空时，点击"立即登录"触发错误提示："请输入用户名或邮箱"
              - 输入不符合上述任一格式时，错误提示："用户名格式不正确（6-30位字母、数字、空格或\"_\"，字母开头）或邮箱格式不正确"
            - 校验不通过时：
              - 输入框添加错误样式（红色边框，颜色为#f5222d）
              - 下方显示错误消息，字体颜色为#f5222d，字号约12px，顶部留白约4px

        1.2.3 密码输入框
            - 类型：密码输入框
            - 占位符："请输入密码"
            - 输入内容实时更新内部状态formData.password
            - UI 样式：
              - 尺寸与用户名输入框保持一致（宽约320px，高约44px，左右内边距约16px）
              - 默认边框与焦点态样式与用户名输入框一致
              - 输入内容以密码点形式显示
            - 校验规则：
              - 密码长度不得少于6个字符
              - 为空时提示："请输入密码"
              - 长度不足6位时提示："密码至少6位字符"
            - 校验失败时：
              - 输入框添加错误样式（边框颜色切换为#f5222d）
              - 下方显示错误消息，字体颜色为#f5222d，字号约12px

        1.2.4 登录按钮
            - 文本："立即登录"
            - 尺寸与样式：
              - 按钮宽度填满表单宽度（100%）
              - 高度约44px
              - 背景颜色为橙色（#ff8c00），文字颜色为白色
              - 字号约18px，字体加粗（font-weight: 600）
              - 无圆角（直角，border-radius: 0）
            - 鼠标交互：
              - hover 时背景加深为#ff7a00，并略微上移（约1px），同时出现橙色半透明阴影，增强点击感
              - 禁用态（请求中）：
                - 背景变为#f5f5f5，文字颜色为#bfbfbf
                - 鼠标指针变为不可点击形态（not-allowed）
            - 点击后行为：
              - 阻止表单默认提交行为
              - 首先执行前端表单校验validateForm
              - 若表单校验不通过，则不发送请求，只更新错误提示
              - 若校验通过：
                - 将按钮置为禁用状态，文本变为"登录中..."
                - 调用loginUser接口发送登录请求（frontend/src/services/auth.ts）

        1.2.5 登录结果处理
            - 请求成功且接口返回success=true：
              - 从响应中读取用户信息与token
              - 调用AuthContext.login(user, token)，写入全局登录状态
              - 弹出"登录成功！"提示
              - 跳转到个人中心页面（/profile）
            - 请求返回错误时，按照后端message进行细分：
              - message === "用户不存在"：
                - 在用户名字段附近展示错误："用户名不存在"
              - message === "密码错误"：
                - 在密码字段附近展示错误："密码错误"
              - message === "账户已被禁用"：
                - 在用户名字段附近展示错误："账户已被禁用，请联系客服"
              - 其他情况：
                - 弹出alert提示：message内容或"登录失败，请重试"
            - 网络或运行时异常：
              - 捕获Error实例，使用error.message通过alert展示
              - 无法识别的异常时统一提示："登录失败，请重试"
            - 请求结束后，无论成功或失败：
              - 解除按钮禁用状态，恢复按钮文本为"立即登录"

        1.2.6 底部链接区域
            - 布局：
              - 整体位于"立即登录"按钮下方，采用水平居中排列
              - 包含"忘记密码？"链接、竖线分隔符"|"以及"注册12306账号"按钮
            - "忘记密码？"：
              - 以a标签形式展示，字体较小（约10px），颜色为#1890ff
              - hover 时颜色加深为#096dd9
              - 点击后通过a标签跳转到"/forgot-password"
            - 分隔符：
              - 使用单独的span元素展示"|"字符
              - 颜色为#ccc，左右各保留约4px 间距，保证可读性
            - "注册12306账号"：
              - 以button形式呈现，背景透明、无边框，仅文字高亮
              - 字体大小约10px，文字颜色为#1890ff，下划线样式增强可点击性
              - hover 时颜色同样加深为#096dd9
              - 点击时调用onNavigateToRegister回调
              - 在App路由中，该回调会跳转至"/register"

        1.2.7 服务说明区域
            - 位于表单底部，通过一条浅灰分割线与表单主体分隔
            - 展示统一服务时间说明文字：
              - "铁路12306每日5:00至次日1:00（周二为5:00至24:00）提供购票、改签、变更到站业务办理， 全天均可办理退票等其他服务。"

    1.3 扫码登录模式

        1.3.1 选项卡切换行为
            - 用户点击"扫码登录"选项卡时：
              - activeLoginTab 状态从"account"切换为"qr"
              - 登录卡片内容从账户表单切换为扫码登录占位区域
            - 用户点击"账户登录"选项卡时：
              - activeLoginTab 切换回"account"
              - 恢复显示账户登录表单

        1.3.2 扫码登录展示内容
            - 二维码占位区域：
              - 使用正方形占位框模拟二维码位置，宽高约216px × 216px
              - 背景为浅色斜线条纹（repeating-linear-gradient），营造占位效果
              - 边框为1px 实线 #ccd6e3，无圆角
              - 与上下内容之间保持约8px-12px 的垂直间距
              - 添加aria-label="扫码登录二维码占位"，用于无障碍说明
            - 提示文字：
              - 紧贴二维码占位区域下方，文案为："打开12306手机APP 扫描二维码"
              - 字号约14px，文字颜色为#555
              - 文案中"12306"四个字符使用高亮色#2a78f6，并加粗显示
            - 底部链接：
              - 下方提供"注册12306账号"按钮，样式为蓝色文字按钮
              - 文本颜色为#2a78f6，鼠标悬停时保持高亮
              - 点击行为同账户登录模式下的注册按钮（调用onNavigateToRegister）
            - 特色说明：
              - 再下方横向展示三段短文案："扫一扫登录"、"更快"、"更安全"
              - 采用flex横向排列，间距约12px
              - 文案颜色为#888，字号约13px，突出扫码登录的优势

    1.4 未实现与差异说明

        1.4.1 与官方12306登录页的差异
            - 当前实现未包含图形验证码输入与验证逻辑
            - 当前实现未包含登录后的二次短信验证弹窗流程
            - 当前实现统一将登录成功后的跳转目标设置为个人中心页面（/profile），而非首页

        1.4.2 后续可扩展点
            - 在登录表单中增加图形验证码输入与刷新逻辑
            - 登录成功后根据用户设置决定是否触发短信二次验证
            - 支持自动登录和记住用户名选项在UI上的展示与持久化行为
